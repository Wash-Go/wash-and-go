# Architecture Decision Records

Use ADRs for decisions that affect system boundaries, durable technology
choices, data ownership, security, or team workflow.

Each ADR records context, decision, consequences, alternatives, and status.
Accepted ADRs override older summaries and proposals.

## Current records

| ADR     | Status       | Scope                                                                      |
| ------- | ------------ | -------------------------------------------------------------------------- |
| ADR-001 | Accepted     | Canonical repository and legacy web preservation                           |
| ADR-002 | Accepted     | React Native mobile, onboarding-only web, and TomTom                       |
| ADR-003 | **Proposed** | Modular monolith, repository boundary, REST/OpenAPI, and conditional Nginx |

**Proposed does not mean approved.** A proposed ADR may guide discussion and a
proof of concept, but contributors must not treat it as a mandatory coding rule
until its status changes to Accepted through review.
