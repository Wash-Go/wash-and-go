# Learning notes

Learning notes turn implementation experience into reusable team knowledge.

For each non-trivial change, explain:

- The concept in plain language.
- Where it appears in Wash & Go.
- Why the chosen approach works.
- A common failure mode.
- How to verify or debug it.
- When an alternative would be more appropriate.

Code-update records may contain the initial learning section. Extract longer or
reusable explanations into this folder and link both directions.
